# Resolved proposals

Shipped, council-passed, closed-by-negative-result, and dated point-in-time
records, relocated here from [`../`](../) to keep the active RFC index focused on
live threads. **Each doc carries its own resolution status in its body** — this
folder is an attic, not a separate lifecycle.

The canonical, status-grouped map is the parent index:
[`../README.md`](../README.md).

These are still under `proposals/`, so the doc-health dead-ref check exempts them
the same way it exempts the parent folder: a resolved proposal's references are
preserved as they were at writing and are not kept current.
